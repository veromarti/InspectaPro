# Functional Use Cases

## 1. Register Company

The system allows a company to register and create its profile within the platform.

This establishes the organizational context for inspections.
## 2. Manage Users

The Company Administrator can:

- Create users
- Assign roles
- Manage technician access

This ensures that only authorized users participate in inspections.
## 3. Define Inspection Types

The Company Administrator can create and manage inspection types.

An inspection type defines the structure and criteria that technicians must follow during execution.
## 4. Create Inspection

The Company Administrator creates an inspection based on a defined inspection type.

The inspection is then prepared for assignment.
## 5. Assign Technician

The Company Administrator assigns a technician to a specific inspection.

This connects operational responsibility to a defined task.
## 6. Execute Inspection

The Technician performs the inspection and submits the results.

The system stores the information for validation.
## 7. Validate Inspection

The Company Administrator reviews submitted results and validates them.

If necessary, corrections can be requested.
## 8. Close Inspection

Once validated, the inspection is closed.

After closure, no further modifications are allowed.
